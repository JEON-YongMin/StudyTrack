# StudyTrack - 스터디 관리 웹 서비스

## 프로젝트 소개

StudyTrack은 스터디 운영과 참여를 보다 효율적으로 관리하기 위한 웹 서비스입니다.  

본 프로젝트는 자바 스터디 이후 학습한 Spring Boot와 JPA를 실제 웹 서비스 구조에 적용해보기 위해 진행한 소규모 팀 프로젝트입니다.  
회원 기능, 스터디 관리 기능, API 문서화 등을 구현하며 웹 서비스 개발 흐름을 경험하는 것을 목표로 했습니다.

## 주요 기능

### 1. 회원 기능

- 사용자 회원가입
- 사용자 로그인
- 사용자 정보 저장 및 관리
- 서비스 이용을 위한 기본 인증 흐름 구성

### 2. 스터디 관리

- 스터디 정보 등록
- 스터디 정보 조회
- 스터디 정보 수정 및 삭제
- 스터디 데이터를 관리하기 위한 기본 CRUD 기능 구성

### 3. API 문서화

- Swagger를 활용한 API 문서 확인
- API 요청 및 응답 구조 정리
- 백엔드 API를 쉽게 확인하고 테스트할 수 있는 문서화 환경 구성

## 개발 과정에서 중점적으로 고려한 부분

### Spring Boot 기반 웹 서비스 구조 이해

Spring Boot를 활용해 Controller, Service, Repository 계층을 나누고, 각 계층이 어떤 역할을 담당하는지 이해하며 프로젝트를 구성했습니다.  
이를 통해 단순한 기능 구현을 넘어 웹 서비스의 기본적인 백엔드 구조를 학습했습니다.

### JPA를 활용한 데이터 처리

JPA를 활용해 데이터를 저장하고 조회하는 구조를 구현했습니다.  
엔티티와 Repository를 기반으로 회원 정보와 스터디 데이터를 관리하며, 객체와 데이터베이스가 연결되는 흐름을 경험했습니다.

### 사용자 인증 흐름 경험

회원가입과 로그인 기능을 통해 사용자가 서비스에 접근하는 기본 흐름을 구현했습니다.  
이를 통해 실제 웹 서비스에서 필요한 사용자 관리 구조와 인증 흐름을 학습했습니다.

### API 문서화

Swagger를 적용해 API 요청과 응답 구조를 확인할 수 있도록 구성했습니다.  
API를 구현하는 것에서 끝나지 않고, 협업자가 쉽게 이해하고 테스트할 수 있도록 문서화하는 과정을 경험했습니다.
