# StudyTrack - 스터디 관리 웹 서비스

## 프로젝트 소개

StudyTrack은 스터디 운영과 참여를 보다 효율적으로 관리하기 위한 웹 서비스입니다.  

본 프로젝트는 자바 스터디 이후 학습한 Spring Boot와 JPA를 실제 웹 서비스 구조에 적용해보기 위해 진행한 소규모 팀 프로젝트입니다. 회원 기능, 스터디 개설·참여, 멤버 관리, 공지·과제 관리 기능을 구현하며 웹 서비스 개발 흐름을 경험하는 것을 목표로 했습니다.

## 주요 기능

### 1. 회원가입 및 로그인

- 사용자는 회원가입을 통해 계정을 생성할 수 있습니다.
- 로그인 후 서비스 이용을 위한 인증 흐름을 제공합니다.

### 2. 사용자 정보 관리

- 로그인한 사용자의 정보를 조회할 수 있습니다.
- 사용자는 닉네임을 변경할 수 있습니다.
- 사용자 정보를 기반으로 스터디 참여 및 활동 흐름을 연결합니다.

### 3. 스터디 개설

- 사용자는 새로운 스터디를 생성할 수 있습니다.
- 스터디 제목, 설명, 최대 인원, 카테고리, 공개 여부 등의 정보를 설정할 수 있습니다.
- 스터디 생성 시 참여자 관리와 초대 기능에 필요한 기본 정보를 함께 구성합니다.

### 4. 스터디 조회 및 상세 정보 확인

- 개설된 스터디 목록과 상세 정보를 확인할 수 있습니다.
- 스터디별 제목, 설명, 카테고리, 공개 여부, 참여 인원 등의 정보를 제공합니다.
- 사용자는 스터디 상세 페이지에서 공지, 과제, 멤버 등 주요 정보를 확인할 수 있습니다.

### 5. 스터디 참여 및 초대

- 공개 스터디에 참여할 수 있습니다.
- 초대 코드를 통해 비공개 또는 초대 기반 스터디에 참여할 수 있습니다.
- 닉네임 기반 초대 기능을 통해 특정 사용자를 스터디에 초대할 수 있습니다.

### 6. 스터디 멤버 관리

- 스터디에 참여한 멤버 정보를 조회할 수 있습니다.
- 스터디 내 사용자 역할을 관리할 수 있는 구조를 제공합니다.

### 7. 공지 관리

- 스터디별 공지를 작성할 수 있습니다.
- 작성된 공지를 수정하거나 삭제할 수 있습니다.

### 8. 과제 관리

- 스터디별 과제를 등록할 수 있습니다.
- 과제 목록을 조회하고, 과제 내용을 수정하거나 삭제할 수 있습니다.
- 과제의 진행 상태, 우선순위 등 학습 관리에 필요한 정보를 다룰 수 있도록 구성했습니다.

### 9. 스터디 설정 관리

- 스터디 정보를 수정할 수 있습니다.
- 스터디 공개 여부, 최대 인원, 카테고리 등 운영 설정을 관리할 수 있습니다.
- 필요 시 스터디 삭제 기능을 통해 스터디를 종료할 수 있습니다.

## 개발 과정에서 중점적으로 고려한 부분

### Spring Boot 기반 웹 서비스 구조 이해

Spring Boot를 활용해 Controller, Service, Repository 계층을 나누고, 각 계층이 어떤 역할을 담당하는지 이해하며 프로젝트를 구성했습니다. 이를 통해 단순한 기능 구현을 넘어 웹 서비스의 기본적인 백엔드 구조를 학습했습니다.

### JPA를 활용한 데이터 처리

JPA를 활용해 데이터를 저장하고 조회하는 구조를 구현했습니다. 엔티티와 Repository를 기반으로 회원 정보와 스터디 데이터를 관리하며, 객체와 데이터베이스가 연결되는 흐름을 경험했습니다.

### 사용자 인증 흐름 경험

회원가입과 로그인 기능을 통해 사용자가 서비스에 접근하는 기본 흐름을 구현했습니다. 이를 통해 실제 웹 서비스에서 필요한 사용자 관리 구조와 인증 흐름을 학습했습니다.

### API 문서화

Swagger를 적용해 API 요청과 응답 구조를 확인할 수 있도록 구성했습니다. API를 구현하는 것에서 끝나지 않고, 협업자가 쉽게 이해하고 테스트할 수 있도록 문서화하는 과정을 경험했습니다.
